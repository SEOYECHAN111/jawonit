# 자원잇다 v33 운영 매뉴얼

## 1. 개인 수거 흐름

1. 개인이 사진 또는 품목명을 입력한다.
2. `disposal_items.json` 1,000개 품목 기준표와 매칭한다.
3. 배출 준비, 주의사항, 즉시수거 기준을 표시한다.
4. 무게·가치·수량·대형품 기준 중 하나라도 충족하면 즉시수거 매칭으로 접수한다.
5. 기준 미달이면 묶음수거 대기 또는 캠페인 참여로 전환한다.

## 2. 기관·기업 입찰방 흐름

- 기관: 온비드 공식 입찰 전후의 공고자료, 현장확인, 반출작업, 내부보고를 위한 참여방을 만든다.
- 기업: 정기수거, 폐기비 절감, ESG 관리 목적의 작업입찰방을 만든다.
- 업체: 입찰방을 열람하고 수거·운반·분류·재활용 작업 제안을 한다.
- 관리자: 전체 입찰방을 열람·관리한다.

주의: 자원잇다는 온비드 공식 입찰 제출, 보증금 납부, 개찰, 낙찰자 결정을 대행하지 않는다.

## 3. 관리자 통합시스템

관리자는 다음 항목을 관리한다.

- 회원·자격·플랜 승인
- 배출 안내표 1,000개 품목 검토
- 품목별 자원시세·고객환급률·물류비·리스크비 설정
- 수익 예상 계산
- 입찰방 생성·상태 관리
- 수거·배차 최적노선 시뮬레이션
- 공지자료 등록
- 캠페인 등록
- 문의 답변
- API 연동상태 점검
- 보호자료 접근권한 관리
- 감사로그 확인

## 4. 배출 안내표 운영 원칙

- 즉시수거: 무게, 가치, 수량, 대형품 기준 충족
- 묶음수거: 단독 수거는 어렵지만 지역 물량을 모을 수 있는 경우
- 캠페인: PET, 박스, 헌옷 등 소액 다량 품목
- 관리자 검토: 위험품, 저장매체, 건설폐기물, 지정폐기물 의심품

## 5. API 키 연결

`.env`에 키를 넣으면 safe-mock 모드에서 실제 연동 모드로 전환할 수 있다.

- `OPENAI_API_KEY`: GPT 서류 자동작성
- `VISION_API_KEY`: 사진 품목 분류
- `OCR_API_KEY`: 공문·목록표 OCR
- `KAKAO_REST_API_KEY`, `NAVER_MAP_CLIENT_ID`: 지도/주소
- `ROUTE_API_URL`: 실제 노선 최적화
- `TOSS_SECRET_KEY`, `PORTONE_API_KEY`: 결제
- `SOLAPI_API_KEY`: 문자/알림톡
- `BIZNO_API_KEY`: 사업자 검증
