# 자원잇다 v37 최종사업 운영 설계서

## 1. 핵심 원칙
- 공개 홈페이지는 개인 고객이 이해하기 쉬운 구조로 유지한다.
- 기관·기업·업체 업무기능은 로그인 후 역할과 플랜에 따라만 노출한다.
- 공식 온비드 입찰 제출, 보증금 납부, 개찰, 낙찰 결정은 자원잇다가 대행하지 않는다.
- 자원잇다는 전후업무 자동화, 내부 참여방, 수거·반출 작업관리, 성과보고, 보호자료 관리를 담당한다.

## 2. 개인 고객 흐름
1. 사진 업로드 또는 품목명 입력
2. 1,000개 배출 안내표에서 품목 매칭
3. 배출 방법, 금지사항, 사진 촬영 기준, 포장 기준 표시
4. 무게·가치·수량·대형품 기준 판정
5. 기준 충족: 즉시수거 매칭
6. 기준 미달: 묶음수거 대기 또는 캠페인 참여
7. 위험품·저장매체·건설폐기물 의심: 관리자 검토
8. 관리자 배차 추천 후 업체 배정
9. 수거 완료 후 정산·성과기록

## 3. 기관·기업 흐름
1. 가입 신청 및 증빙자료 제출
2. 관리자 승인
3. 입찰방 생성 또는 AI 서류작성 이용
4. 업체 제안서 접수
5. 가격·일정·차량·안전·보안·재활용계획 평가
6. 작업관리 및 성과보고

## 4. 업체 흐름
1. 가입 신청 및 허가·차량·보험자료 제출
2. 관리자 승인 및 플랜 선택
3. 입찰방 열람
4. 제안서 제출
5. 관리자 또는 발주기관/기업 검토
6. 작업배정, 수거·반출, 정산

## 5. 관리자 통합시스템
- 회원 심사대기열
- 회원·역할·플랜·신뢰점수 관리
- 1,000개 배출 안내표 검색·수정
- 품목별 단가정책 및 수익 계산
- 입찰방 생성·상태관리
- 제안서 평가·선정
- 개인 수거신청 배차 추천
- 공지·캠페인 운영
- 문의답변
- API 연동센터
- 최종 출시준비 점검
- 운영자료 JSON 내보내기

## 6. Render 배포 후 필수 작업
- SECRET_KEY 환경변수 교체
- 관리자 비밀번호 변경
- OpenAI, OCR, Vision, 지도, 결제, 문자 API 키 입력
- 실제 사업자 정보, 통신판매업, 주소, 연락처 반영
- 폐기물 수집·운반·처리 허가 및 개인정보 파기 업무 법률 검토
- PG 심사 및 환불정책 최종 검토
- S3 또는 외부 파일저장소 연결
- PostgreSQL 전환 권장

## 7. 현재 상태
이 버전은 API 키가 없으면 safe-mock으로 동작하며, Render 환경변수에 키를 넣으면 실연동 준비 상태로 전환되도록 설계되어 있다. 실제 법률·허가·PG·개인정보 위탁계약은 사업 전 별도 완료해야 한다.
